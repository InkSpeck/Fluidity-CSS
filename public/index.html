Grid System

Grid Config Variables:

Located and configured at the top of lib/grid.styl, the gridWidth and minWidth variables are used determine the fluidity of the grid.

Stylus
// Set the width of the grid in % or px // Set the minwidth of the grid in px gridWidth = 100% minWidth = 320px

For example a gridWidth of 100% and a minWidth of 320px would allow the layout to expand to the full width of any viewport and shrink down to the mobile friendly width of 320px, no smaller.

A Fixed Width layout can be achieved by setting both gridWidth and minWidth to the same width in px. Though... why would you want that?

The Grid()

To use the grid in your layout, you'll need to create a container element (a div) for your layout and then simply apply the grid to your container either by using the grid() mixin on your semantically named container within your stylus or by using the .grid class on your container.

Stylus
#about_page grid()
HTML
<div class="grid"> Content... </div>

Either of these methods can work to apply the grid. But of course, semantic is better.

Grid Rows:

elements

<header> / <section> / <fieldset> / or as a class <div class="section">

Once the grid is applied to the container you can begin constructing your grid with section elements and container elements. The <section> element essentially functions as a row in your grid. If your not using HTML5 (Why aren't you?) you may also use a <div> with a class of .section. The <header> element and the <fieldset> element also behave as a row/section and do not require a parent section.

Grid Columns:

mixins

grid_10( ) / grid_20( ) / grid_25( ) / grid_30( ) / grid_33( ) / grid_40( ) / grid_50( ) / grid_60( ) / grid_70( ) / grid_75( ) / grid_80() / grid_90( ) / grid_100( )

Containers within these sections can be given their column widths by using the appropriate grid_Width%() mixin or by using a .grid_Width% class in your HTML. These mixins define the column widths of your grid elements. With these combinationations virtually any configuration you can dream up can be created. Each is also available as class for use directly in your non-semantic html. This can make prototyping a layout quick and then later they can made semantic.

A Semantic Grid Example:

An example of a 2-column layout with a header and footer.

HTML
<!DOCTYPE HTML> <html> <head> ... </head> <body> <div id="examplepage"> <header> Content... </header> <section id="content"> <sidebar> Content... </sidebar> <article> Content... </article> </section> <section> <div id="footer"> Content... </div> </section> </div> </body> </html>

View this example with some content and color added for clarity.

Stylus
@import "fluidity/fluidity" #examplepage grid() #content sidebar grid_30() article grid_70() #footer grid_100()
